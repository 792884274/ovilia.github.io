The simple types of JavaScript are numbers, strings, booleans (true and false), null, and undefined. All other values are objects.

Numbers, strings, and booleans are object-like in that they have methods, but they are immutable. Objects in JavaScript are mutable keyed collections. In JavaScript, arrays are objects, functions are objects, regular expressions are objects, and, of course, objects are objects.

--JavaScript: The Good Parts

Numbers, strings, booleans, null, and undefined are called primitive types.

typeof VS instanceof

Here's something about typeof operator.

    typeof undefined;       // "undefined"
    typeof true;            // "boolean"
    typeof 1;               // "number"
    typeof "";              // "string"
    typeof {};              // "object"
    typeof null;            // "object"
    typeof function(){};    // "function"
    typeof [];              // "object"

Note that typeof null is "object" rather than "null". But null is primitive type since null instanceof Object returns false.

Another thing you should pay attention to is that although function and array are both object types, typeof a function is "function" while typeof an array is "object". Professional JavaScript for Web Developers gives us a possible explanation.

Technically, functions are considered objects in ECMAScript and don’t represent another data type. However, they do have some special properties, which necessitates differentiating between functions and other objects via the typeof operator.

--Professional JavaScript for Web Developers

One thing that typeof cannot tell is what type of the object it is. So ECMAScript provides the instanceof operator for reference values. Basic use is like:

    function A() {};
    var a = new A();
    a instanceof A;     // true

All reference values, by definition, are instances of Object, so the instanceof operator always returns true when used with a reference value and the Object constructor. Similarly, if instanceof is used with a primitive value, it will always return false because primitives aren't objects.

--Professional JavaScript for Web Developers

So, it's not surprising to find: 

    function A() {};
    A instanceof Object;     // true since function is object type
    
    null instanceof Object;  // false since null is primitive

Another interesting thing about instanceof is when it meets with Boolean and String. Boolean and String are the reference types of boolean and string respectively. Note the difference since they are reference types:

    var aStr = "abc";
    aStr instanceof String;             // false
    
    var aStrObj = new String("abc");
    aStrObj instanceof String;          // true

It's very important to understand the difference between a primitive Boolean value and a Boolean object — it is recommended to never use the latter.

--Professional JavaScript for Web Developers

Properties

Sometimes, you may find the || operator useful for the default value if a property of an object doesn't exist.

    var a = {
        x: 3.2,
        y: 3
    };
    var z = a.z || 0;   // set z to 0 if a.z is undefined

And you shall always check if a property exists before use.

    var a = {};
    a.z;                // undefined
    a.z.i;              // throw "TypeError"
    a.z && a.z.i;       // undefined

Reference

Objects are passed by reference without being copied.

    var a = {}, b = {}, c = {}; // a, b, and c each refer to a
    a = b = c = {};             // a, b, and c all refer to the same empty object

Delete

The delete operator can be used to remove a property from an object. It will remove a property from the object if it has one. It will not touch any of the objects in the prototype linkage. Removing a property from an object may allow a property from the prototype linkage to shine through.

--JavaScript: The Good Parts

Let me show this in the following example.

    function A() {};
    A.prototype.b = 'b in prototype';
    
    var a = new A();
    a.b = 'b in a';     // a.b point to 'b in a'
    
    delete a.b;         // a.b point to A.prototype.a
    a.b;                // returns 'b in prototype'

You may expect a.b returns undefined on the 8th line, but actually it returns 'b in prototype'. This is because it will not delete the element in prototype linkage. And when you retrieve a.b on the 8th line, it will first look in object a and not find, then look in prototype chain and find it in A.prototype.b so returns 'b in prototype'.

Global variables

One way to minimize the use of global variables is to create a single global variable for your application that variable then becomes the container for your application.

        var MYAPP = {};
        MYAPP.stooge = {
            "first-name": "Joe",
            "last-name": "Howard"
        };
